# Architecture: Agentic Strategic Command Center

The Agentic Strategic Command Center is built on a **Metadata-Driven Architecture** designed to function as a domain-agnostic "Operating System" for strategic execution. 

Rather than hard-coding business logic, the system ingests a central JSON configuration to define its environment, and then delegates execution to a specialized **Multi-Agent Ecosystem**.

## 1. Core Architecture: The Metadata Engine

The foundation of the application relies on complete decoupling of the UI from the business domain. The app ingests a `config.json` payload containing:
*   **Domain Configuration:** Defines the industry, compliance standards, and active external signals.
*   **Strategic Roadmap:** The current features in the pipeline and their base prioritization.
*   **Audit Rubrics:** The specific constraints (e.g., Clinical Accuracy, OWASP Top 10) the system must adhere to.

By hot-swapping this JSON file, the entire application can instantly pivot from a Health-Tech context (ABDM compliance) to an Enterprise SaaS context.

## 2. The Multi-Agent Ecosystem (The "Agentic" Core)

The application moves beyond passive dashboards by automating the **OODA loop** (Observe, Orient, Decide, Act) through four specialized, interconnected simulated agents:

### The Scout Agent (Intelligence Ingestion)
**Role:** The Observer.
*   Continuously monitors and ingests external signals from the environment (e.g., regulatory changes, competitor sentiment, dark web telemetry).
*   Quantifies qualitative data by assigning an `intensity` score to each incoming signal, pushing it to the global state radar.

### The Strategist Agent (Signal-to-Priority Matrix)
**Role:** The Orienter & Decider.
*   Listens for state changes triggered by the Scout Agent.
*   Utilizes a dynamic re-ranking algorithm to evaluate the roadmap against incoming signals.
*   Autonomously shifts roadmap priorities (e.g., escalating an Identity Vault feature to "Urgent") when a high-intensity signal breaches the threshold, instantly mitigating "Data Blindness" and feature waste.

### The Architect Agent (Vibe Coding)
**Role:** The Actor (Builder).
*   Triggered by the Strategist's prioritization shifts.
*   Functions as a "Vibe-to-Code Accelerator." It utilizes generative AI to autonomously write functional boilerplate, security scripts, or API schemas tailored to the exact requirements of the new high-priority signal.

### The Sentinel Agent (LLM-as-a-Judge)
**Role:** The Auditor (Quality Gate).
*   Acts in parallel with the Architect Agent.
*   Consumes the output generated by the Architect and scores it against the strict `judge_rubrics` defined in the metadata core.
*   Validates outputs for domain-specific constraints like "Clinical Faithfulness" or "Professional Documentation Protocols," ensuring zero compliance penalties before any code is approved.

## 3. Technology Stack & Data Flow

*   **Frontend Ecosystem:** React 18 & Vite (for rapid, lightweight component rendering).
*   **Data Flow:** Unidirectional, event-driven state flow. A user (or automated webhook) triggers the **Scout Radar** -> Cascades to the **Strategist Prioritizer** -> Triggers the **Architect Accelerator** -> Scored by the **Sentinel Terminal**.
*   **Styling:** Pure CSS enforcing a "Cybersecurity-grade" high-contrast aesthetic, ensuring high information density without visual clutter.
